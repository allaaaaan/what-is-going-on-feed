今天是二零二六年五月十四日，「又咋啦?!」完整版，今天三十三条值得关注的消息，分五个方向来讲。

【AI 模型与能力】

先从一个基础性的研究发现说起。所有人训练大模型都在用 token 作为计算单位——token 是文本被分词器切割后的最小处理块——但研究人员对超过一千三百个模型做了系统对比实验，发现用字节，也就是原始文本最小的存储单元，不经过分词，来度量训练量，效果更准、跨语言泛化更稳定。Scaling law 是基础模型团队预测"加多少算力、产生多少能力"的核心公式，如果计算单位用错了，预测就会系统性偏差。更直接的影响在非英语语言：现有分词器对中文、阿拉伯语、印地语等语言的处理效率明显低于英语，字节层面处理绕过了这个不均等，理论上能给这些语言带来免费的质量提升。这是一个涉及 LLM 训练经济学根基的发现，而不是一个增量优化。

模型效率方向有两个独立结果，放在一起看有意思。第一，用强化学习（RL）对 4B 参数的小语言模型做微调，多项评测上可以追平 Claude Sonnet 4.6 这类参数量是它十倍以上的生产模型。RL 微调的核心逻辑是：不靠更大的参数量，而靠更精准的奖励信号来提取模型的潜在能力。第二，Google 把 Gemini 蒸馏到了 2600 万参数的 Cactus Needle 模型，可以在普通 Mac 或 Windows 笔记本上以生产速度运行。两条合在一起说明：模型能力的前沿正在向小型化、本地化扩散。对做大量 inference 的团队，"买 API"还是"自部署"的那个经济临界点在移动；对做端侧产品的团队，原本不成立的产品假设可以重新验证。

Serverless GPU 推理的可行性，这周有了具体的工程支撑。AI 基础设施云平台 Modal 发布了工程博客，记录了如何把 GPU 冷启动延迟从多秒压到几十毫秒。过去，serverless 和 GPU 推理在实践中很难兼容，核心障碍就是冷启动：用户发请求，等几秒 GPU 才准备好，交互场景根本跑不了。现在这个约束松了。对做 AI 产品的团队，变化体现在两个地方：infrastructure 账单从固定（需要一直跑着的实例）变成按请求计费；以前因为启动成本太高放弃的低频 AI 功能，现在有经济账了——比如按需触发的深度分析、不常用的 AI 助手模式，不必为峰值容量预购。

本周 AI 产品层几条新动态。Claude Opus 4.7 上线快速推理模式（Fast Mode），在 API 和主流代码编辑器里开放预览，对开发者是更快、更便宜的交互推理。Meta 的 Muse Spark 模型开始在 Meta 全家桶和 Ray-Ban 智能眼镜里部署，提供实时摄像头识别和语音助手。Qwen 发布了 Qwen-Image-2.0，图像生成在版式排版、真实感和长文本渲染上有提升，对依赖图像生成的产品团队有参考价值。还有一家叫 Perceptron 的公司发布了视频分析 AI 模型，定价比 Anthropic、OpenAI、Google 同类产品低 80% 到 90%——在视频 AI 基础设施评估上，多了一个成本竞争选项。

最后，Parameter Golf 比赛——让参与者在参数量约束下优化机器学习模型的竞技活动——今年超过一千人参加，AI 编程代理作为独立参赛者拿到了排名靠前的成绩。这是研究竞赛里 AI 代理开始以独立能力参与、而不只是辅助工具的早期案例，信号意义大于实际影响。

【AI 产业与商业】

这个方向今天有几条拼在一起，能讲同一个主题：AI 原生公司是什么样的，它和传统 SaaS 公司的商业逻辑区别在哪里。

先看一条市场数据。Anthropic 今年在企业 AI 市场的份额增长超过一倍，OpenAI 虽然仍以 56% 居首，但份额下降了 8%，Google Gemini 增长到 40%。企业 AI 供应商格局正在快速流动，选择尚未定型。

然后是两条关于 AI 原生公司的分析。

第一条：AI 原生软件正在崩解 SaaS 的毛利结构。传统 SaaS 的毛利率大约 70%，核心逻辑是代码写一次、分发成本接近零。但 AI 原生产品的成本结构完全不同：个性化 AI 响应无法缓存，每次调用推理模型消耗 10 到 100 倍的 token 量。文章给出了一个具体数字：月活百万的产品，如果每次用户交互都走推理模型，毛利率从 70% 掉到 17%。这不是边际调整，是商业模式层面的结构性变化。对 SaaS 创始人来说，按传统方式定价、按 AI 原生方式建产品，账是对不上的。解法要么是重新定价（跟 usage 走），要么是找出哪些场景可以用缓存或更轻量的模型替代完整推理。

第二条：AI 原生转型不是"把 AI 功能嵌进去"，而是重建工作流和知识结构，让 AI 代理能作为业务流程的核心节点运行，而不是外挂工具。文章区分了两种模式：一种是"给员工 AI 工具，让他们探索"；另一种是"把业务流程的输入输出显式文档化，让代理能独立执行"。后者门槛高，需要组织有足够的文档积累和数据结构化能力，但带来的是杠杆效应，而不只是效率提升。文章判断：5 百万美元 ARR 以上的公司，投资后者开始有回报；更早期的，工具化够用。

GitHub Copilot 这周调整定价：Pro 和 Pro+ 计划提供更多使用量，同时推出面向高频使用的 Max 计划。在 AI 编程工具竞争激烈的背景下，这是一次定位调整，意在留住重度用户，防止流向 Cursor、Claude Code 等替代品。

Amazon 内部 AI 工具的使用数据出了一条有意思的故事：公司推了一套 AI 工具使用排行榜（代号 MeshClaw），员工为了在榜单占位，开始执行不必要的任务刷数据。这是经典的 Goodhart 定律案例——当指标变成目标，指标就失效了。更深的安全影响是：当 AI 代理能代表员工执行操作，"刷任务"的行为等于给了代理宽泛的授权边界，这是一个主动暴露安全风险的操作，而不只是浪费。

【太空与基础设施】

SpaceX 这周有两条消息，放在一起看背后是同一个方向。第一，Starship 第三版达到四百零八英尺高，是人类建造过最高的火箭，新引擎加上在轨燃料加注能力，单次有效载荷上限再次扩大。第二，SpaceX 正在规划全球星港网络，目标是支持每年数千次发射，发射场地理上分散部署于全球。两条合在一起说明的是：商业航天正在从稀缺资源走向可规模化基础设施，进入轨道开始变成一个相对标准化的操作，而不是每次都是大事件。

轨道数据中心方向，Google 和 SpaceX 正在讨论把 AI 推理硬件放到太空，服务卫星星座和边缘网络。逻辑是：地面数据中心受电力和散热制约，而太空环境天然有更好的散热条件和更少的基础设施约束。这个方向仍处早期，实际落地周期在年级以上，但方向正在从讨论变成规划。

半导体供应链有一条需要留意：模拟芯片和电源芯片厂商正在把产能优先级从电动车和太阳能转向 AI 基础设施，因为 AI 这端毛利更高。直接后果是汽车和可再生能源行业面临供货紧张和价格上涨。AI 基础设施建设对其他行业的芯片供给形成了挤占效应，这个影响还在持续扩大。

MIT 研究人员开发了一种能在海水电解制氢的腐蚀性环境中存活的不锈钢材料，解决了沿海制氢的一个主要工程障碍——专用耐腐蚀合金太贵。如果这个材料能进入量产，海边绿氢的成本结构会改变，大规模部署变得更现实。绿氢目前最大的几个门槛：电力成本、材料耐久性、运输，这条消息敲掉了其中一个。

加利福尼亚州一家主要电力公司正在考虑把原来服务太浩湖五万名居民的输电线路，改道供给数据中心。AI 基础设施的电力需求和居民用电之间出现了直接的资源竞争。数据中心选址的隐性社会成本正在变成公开讨论的议题，监管层面的反应还不确定，但这类冲突多了以后，会倒逼政策介入。

【消费科技与平台】

Google 宣布了 Googlebooks，运行 Android 系统的 AI 笔记本，内置 Gemini，能感知屏幕上下文、接受手势指令。这不是 Chromebook 的升级，而是一个新品类——AI 原生的 PC，在市场定位上直接跟 Windows 笔记本和 Mac 竞争。对消费者来说短期内很难判断体验是否成立，但品类的存在本身改变了市场格局。值得观察的是：Android 应用生态在生产力场景的深度还不如 Windows 或 macOS，这是 Googlebooks 需要解决的硬约束。

Android 的 AI 代理功能整合，通过 Play Services 而不是系统升级推送，覆盖现有 20 亿设备。开发者视角：如果 Gemini 能跨 app 完成任务，以前需要用户手动在多个 app 间切换的场景，可能会被折叠进一个对话框。对 app 开发者来说，这是双向的：被 Gemini 调用是分发机会，但用户不再直接打开你 app 是流量风险。Gemini 代理的能力边界、调用授权机制，会是接下来几个季度开发者最需要看清楚的。

Apple 宣布将在 6 月 8 日的 WWDC 上发布 iOS 27，更新包括可定制化的相机应用、重新设计的 Siri、以及全系统的视觉和动效变化。开发者需要关注的是设计模式调整——新的系统级 UI 风格通常需要 app 做适配，才能保持视觉一致性。从 Apple 过去的 WWDC 节奏来看，重大 UI 改动一般带来 3-6 个月的开发者适配窗口。

Bambu Lab 3D 打印机的固件锁定问题有了社区进展：一个 Pull Request 被提交，要求恢复对局域网的完整本地控制，绕过制造商强制云端才能完整工作的限制。这是消费者硬件所有权的典型冲突——你买了设备，但制造商通过固件控制用途边界。PR 被合并不等于问题解决，但用户侧的压力在积累，类似的博弈已经在打印机、智能家居、路由器等多个品类里发生过。

Python 3.14 和 3.15 版本将回退 3.13 引入的增量垃圾回收器，原因是性能表现不稳定。对依赖即将发布版本的团队，之前基于增量 GC 的性能预测需要重新评估，替代的优化路径还没有官方定论。

Windows 游戏 API 的实现正在直接进入 Linux 内核，Linux 游戏性能开始有机会真正接近 Windows 水平。对游戏开发者来说，这是平台格局变化的信号，虽然短期内不影响主流发行策略，但长期降低了 Linux 作为游戏平台的摩擦。

【平台权力与数字主权】

支付基础设施作为平台治理工具，这周有一个清晰的案例：支付处理商正在向 Kickstarter 施压，要求封禁成人内容，否则断支付渠道。Kickstarter 的内容政策不是 Kickstarter 自己做的决定，而是被支付链路倒逼的。这已经是这类案例的第三次重演——Patreon、OnlyFans 都出现过相似情形。背后的结构是：Stripe、Visa、Mastercard 等支付处理商的上游是发卡银行，发卡银行有内容合规要求，这个要求沿着链路传下来，压在平台上，而不是压在内容政策制定者身上。创作者和平台都没有反制杠杆，除非能绕开传统支付体系。

GameStop 对 eBay 发出五百六十亿美元敌意收购要约，被董事会以"缺乏可信度"为由拒绝。从资本结构看，GameStop 没有明显的融资能力支撑这个规模，拒绝是预料之内。值得想一想的背景：两家都在走下坡路的电商平台，以规模合并为由发起要约，是一种常见的"最后手牌"策略——向股东展示还有战略选项，而不是真的有执行路径。eBay 用"缺乏可信度"回应，说得相当直白。

FreePascal 项目从 GitHub 迁移到了自托管的 Forgejo——一个开源 Git 代码托管平台，可以理解为自己能部署的 GitHub 替代品。这条消息本身影响范围不大，但是开源社区里"迁出 GitHub"动作慢慢积累的信号之一，动因是对微软/GitHub 的平台依赖和政策走向的担忧。

两名双胞胎兄弟，在被解雇后的几分钟内删除了 96 个政府数据库。这是一个极端的内部威胁案例，但它暴露了一个 IT 管理里常被忽视的时间窗口：解雇通知到权限撤销之间的这段时间，是高风险窗口。任何拥有生产系统访问权限的员工，在离职流程中如果权限没有即时撤销，就是一个潜在的破坏窗口。对运维和安全团队来说，这应该触发一个具体的问题：我们的账号注销时序，能在员工被通知解雇的同一时刻自动撤销权限吗？

最后是欧洲技术主权。Hacker News 上那篇个人基础设施迁移记录拿到了七百一十八分，高分背后反映的是一个更广泛的情绪：在数据主权和地缘政治层面，越来越多的开发者和机构把对美国大厂云服务的依赖视为一种风险，正在主动评估替代方案。欧洲的选项是存在的——Hetzner 做服务器、Infomaniak 做云服务、Proton 做邮件和存储——但整合度和生态规模远不如 AWS、GCP、Azure。原文没有展开，根据现有公开信息推断：短期内欧洲替代方案更适合特定合规场景（GDPR 严格场景、政府项目、金融机构），而不是全面替代主流云。这个市场的成熟需要时间，但需求端的转变已经在发生。
